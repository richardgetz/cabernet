Cabernet

This is a list of interfaces provided by Cabernet. If it states HDHR, then the interface matches that of HDHomerun tuners.

  • /device.xml: DLNA/SSDP device description response
  • /discover.json: HDHR, app metadata
  • /lineup_status.json: HDHR, stream source data
  • /lineup.json: HDHR, channel list in JSON format
  • /lineup.xml: HDHR, channel list in XML format
  • /xmltv.xml: XMLTV formatted EPG
  • /channels.m3u: M3U formatted channel list
  • /playlist M3U formatted channel list
  • /config.json: JSON list of most config parameters
  • /watch/#id: Tunes stream to Channel ID requested
  • /auto/v#id: HDHR, Tunes stream to Channel ID requested
All links above have the same ability to apply to any plugin. You can request the data filtered for a plugin or for an instance of the plugin. As an example: 
    http://idaddress:6077/PlUtotV/DeFauLt/channel.m3u
will generate a list of channels for the PlutoTV:default instance while 
    http://idaddress:6077/M3U/channel.m3u
generates a list of channels for all instances of the M3U plugin, combined.

Examples of URLs

To get an EPG for everything under XUMO 
    http://idaddress:6077/Xumo/xmltv.xml
To get a M3U formatted channel list of M3U:Stirr 
    http://idaddress:6077/m3u/sTirR/playlist or
    http://idaddress:6077/m3U/sTirR/channels.m3u
To get EPG for all plugins in Cabernet 
    http://idaddress:6077/xmltv.xml
For TVHeadend, it would be appropriate to create a network for each plug or plugin:instance.
For Plex, it needs a unique IP address for each tuner, so either give it all plugins at once, or create multiple IP Addresses to separate the plugin data.
For Emby and JellyFin, you can create a TV Source and Guide Data for all, per plugin, or plugin:instance.
